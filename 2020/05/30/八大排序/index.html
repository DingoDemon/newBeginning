八大排序

发表于 2020-05-30

前言

八大内部排序按时间复杂度分别为

  • 插入排序
  • 选择排序
  • 冒泡排序
  • 希尔排序
  • 归并排序
  • 快速排序
  • 堆排序
  • 基数(桶)排序

时间复杂度/空间复杂度与是否稳定排序关系如下

时间复杂度 空间复杂度 是否稳定排序
插入排序 O(n^2) O(1) 是
选择排序 O(n^2) O(1) 否
冒泡排序 O(n^2) O(1) 是
希尔排序 O(n^(1.3~2)) O(1) 否
归并排序 O(n*logn) O(n) 是
快速排序 O(n*logn) O(1) 否
堆排序 O(n*logn) O(1) 否
基数 N*(log(N/M)+1) O(m+ n) 是

冒泡排序

从 index = 0 开始，每一位和下一位这一对之间比较大小，如果后者小于前者，则交换位置。重复这个过程直到 index = 数组length-1，也就是最后一对。

并且，每次过程都会将最大值放在队列尾部。所以内部循环不需要到ints最尾部，而是数组length-index。

并且，在一次过程中，没有进行操作的话，证明已经有序。不需要下次循环了

整个过程，数组从最后位开始有序

快速排序

  1. 在数列中，取一个基准(pivot) 比如第一次为ints[0];

  2. 重新排序数列，所有元素比基准值小的摆放在基准前面，所有元素比基准值大的摆在基准的后面（相同的数可以到任一边）。在这个分区退出之后，该基准就处于数列的中间位置。这个称为分区（partition）操作。

  3. 递归地对大于/小于两部分执行这个操作。

快速排序的优化

如果使用未优化的快速排序，对一个有序数组进行排序，有可能产生栈溢出风险

因为pivot会为1->递归，2->递归，3->递归，4->递归。

对于小数组来说，快排是慢于插入排序的。

堆排序

堆排序是指利用堆这种数据结构来进行排序。

因为前提为从小到大有序，所以使用小根堆。

堆是一种特殊的树，只要满足下面两个条件，它就是一个堆：

  • 条件一: 堆是一颗完全二叉树。

完美二叉树是指所有层都达到最大节点数的二叉树

完全二叉树是指除了最后一层其它层都达到最大节点数，且最后一层节点都靠左排列。(可以不完美，但得靠左排列)

完满二叉树是除了叶子结点之外的每一个结点都有两个孩子结点。

  • 条件二: 堆中某个节点的值总是不大于（或不小于）其父节点的值。

所以可以用一位数组来表示一个堆，数组index = 0 默认不放数据。所以根节点的位置是1，那么一个位置为k的节点，两个字节点分别为2k和2k+1。同样的，一个位置为k的节点，父节点位置为k/2。

如何构造一个堆

插入元素

将元素放入数组尾部。这样，就满足条件一，但不满足条件二，于是进行上浮。

插入的节点与它的父节点相比，如果比父节点小，就交换它们的位置。直到比父节点大或成为根节点。

删除堆顶元素

将 index = 1 的元素移除。然后，将最后一个元素，移到根节点上。并且还让他满足条件二，所以需要进行下漏。

下漏即和子节点中较小的那个进行交换，直到没有左右子节点。交换结束。

各类排序算法使用场景

  1. 如果输入数据较少(<32),使用插入排序
  2. 如果本身有序或大部分有序，使用冒泡
  3. 如果输入数据较大，使用时间复杂度为N*logN的排序算法，如果对稳定有要求，使用归并，对空间敏感使用快排or堆排
