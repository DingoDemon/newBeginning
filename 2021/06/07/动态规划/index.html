首先感谢题目总结

Leetcode 题解 - 动态规划

完成了所有leetcode题目

刷完之后的提升在于能建立dp流程的感知能力

  1. 模拟流程打表
  2. 推导递推方程

然而，路漫漫兮其修远，这仅仅是一个开始

Fib

斐波那契数列作为动态规划的入门题目是很好理解的

fib(5) = fib(4) + fib(3)

fib(4) = fib(3) + fib(2)

fib(3) = fib(2) + fib(1)

可以看到在计算过程中有大量重复计算，所以需要hashMap或者数组，来对已经完成的计算，进行O(1)的读取操作

打家劫舍 leetcode 198题

错误思想

回溯，从第零或者第一家开始偷。要不跳过1家，要不跳过2家，直到最后一家。

时间复杂度 O(2^n)

超时 fail

正确思想

定义dp数组来规划抢到第i家时的最大金钱。要么是上上家的钱加上当前家，要么是上一家

dp[i] = max(dp[i-2] + house[i] , dp[i-1])

时间复杂度O(n),空间复杂度O(n)

##打家劫舍II leetcode 213题

一共i家，首位相邻，也就是说，抢了第一家就不能抢最后一家，抢了最后一家，就不能抢第一家。可以划分为两个集合

  • 0 ～ i-2
  • 1 ～ i-1

矩阵的最小路径和 leetcode 64题

只能向右or向下走一步，那么，除了第一行or第一列，都是由上或者左移步而来

也就是走到grid[i][j]格子后，

dp[i][j] = min(dp[i-1][j] , dp[i][j-1]) + grid[i][j]

矩阵的总路径数 leetcode 62题

其实就是二维fib，每一步都有左or上转移而来

等差数列划分 leetcode 413题

我承认有赌的成本,打表成功的。位置i自身能组成等差数列，加上i-1的，即为count

时间复杂度O(n^2)，空间复杂度O(1)但是，这实际上是暴力解

实际上，如果能构成等差数列，那么dp[i] = dp[i-1] + 1

然后累加即可

官方解为时间复杂度O(n)，空间复杂度O(1)

分割整数的最大乘积 leetcode 343题

重点是如何得到dp方程

可以进行fori ，讲数字拆分为 i 和n-i

将 n 拆分成 i 和 n-i 的和，且 n-i 不再拆分成多个正整数，此时的乘积是 i × (n - i)

将 n 拆分成 i 和 n-i 的和，且 n-i 继续拆分成多个正整数，此时的乘积是 i * dp[n-i]

最小为 1 * n-1。

按平方数来分割整数 leetcode 279题

一定是有解的，因为n可以拆分为n个1相加，所以，初始化数组

dp[i] = i

然后用只考虑 i^2<=n 的基数

转移方程:

dp[i] = min(dp[i-基数平方]+1 , dp[i])

分割整数构成字母字符串 leetcode 91题

如果char[i] 位，不为’0’，那么可以被理解为新的1位，所以此处要加上dp[i-1]
如果char[i] 位，i>=2,并且能和前面一位组成11～26的话，那么至少可以可以被理解为新的2位，所以此处要加上dp[i-2]

这，其实也是fib的变种

最长递增子序列 leetcode 300题

转移方程

dp[i] = dp[j] +1 ，其中 0 <= j < i 并且 num[j] < num[i]

也就是，如果将num[i]添加到队列尾部。还是升序的。那么就可以在之前队列的基础上+1

时间复杂度O(n^2)
本题还可用2分进一步优化，2分题目总结再看

一组整数对能够构成的最长链 leetcode 646题

跟上道题类似，可以说是换皮题，先排序即可

最长摆动子序列 leetcode 376题

吃了些苦头才做出来，第i个数都要和i-1进行对比。

最长公共子序列 leetcode 1143题

很经典的一道题，从第一次打表到想清楚，以及想清楚 leetcode1035. 不相交的线题怎么换的皮。过程很愉悦

递推方程

如果char[i] 等于char[j]

dp[i][j] == dp[i-1][j-1] +1

否则

dp[i][j] == max(dp[i-1][j] , dp[i][j-1])

背包问题概述

划分数组为和相等的两部分 leetcode 416题

可以转化为一个背包大小为 sum/2 的 0-1 背包问题。

压不成1维背包白学了！

目标和 leetcode 494题

首先，这道题我是回溯完成的

然后这道题，我其实有难度,可以转换成01背包问题

先对nums求和得到sum，因为都是正整数，所以不用考虑绝对值的问题

添加正负号的，之后，取值范围就是[-sum,sum] 创建dp数组dp[i+1][2*sum+1]

对第i个元素，达成和 x 的个数

dp[i][x] = dp[i-1][x - nums[i]] + dp[i-1][x + nums[i]]

其中

x - nums[i] >= -sum

x - nums[i] + sum <= 2 * sum

其实条件 dp[0][sum] = 1

最终答案就是dp[nums.length][sum + target]

01字符构成最多的字符串 leetcode 474题

比较单纯的二维背包问题

dp[i][j][k] = max(dp[i-1][j][k] , dp[i-1][j-[ai]][k-b[i]]+ v)

压缩成二维数组的话

dp[j][k] = max (dp[j][k] , dp[j-[ai]][k-b[i]]+ v)

找零钱的最少硬币数 leetcode 322题

完全背包问题,见注释

找零钱的硬币数组合 leetcode 518题

一样的完全背包问题，对于组合个数

dp[i] += dp[i-num]

字符串按单词列表分割 leetcode 139题

看完题目，也是完全背包问题。但是因为想到要裁剪字符串，想复杂了。正确的方法是用boolean dp数组，储存0～s.length范围内每一位是否可以被正确解构

也就是

dp[i] = check(0~i)||check[i-j]&&dp[j]

dp[i] += dp[i-n]

需要冷却期的股票交易 leetcode 309题

一天结束时一共三种状态，持有股票，不持有且在冷静期，不持有，且不在冷静期

  • 第一种，需要考虑，是当天买的，还是不在冷静期后买的

  • 第二种，肯定是现在卖了，也就是前一天持有的总收益加上当前价格

  • 第三种，手上为空，考虑前面2，3状态的最大值

只能进行两次的股票交易 leetcode 123题

还是二维数组[i][0~3]

  • 0 表示第一次买
  • 1 表示第一次卖
  • 2 表示第二次买
  • 3 表示第二次卖

注意，初始化时，买了，卖，再买，所以 dp[2] = dp[0] = -prices[0]
