数组和字符串

LC 三数之和

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请

你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

先进行排序，相同数字直接跳过，后续就不用去重了。为每一个数在剩余数组中找到起相反数之和

LC 矩阵置零

给定一个 m x n 的矩阵，如果一个元素为 0 ，则将其所在行和列的所有元素都设为 0 。请使用 原地 算法。

一个直观的解决方案是使用 O(mn) 的额外空间，但这并不是一个好的解决方案。

一个简单的改进方案是使用 O(m + n) 的额外空间，但这仍然不是最好的解决方案。

你能想出一个仅使用常量空间的解决方案吗？

  1. 不能直接修改矩阵，因为后续无法知道0是天生的还是后修改的。

  2. O(m+n) 指的用2个boolean[]数组储存该行/列是否出现过0，再次遍历时 如果boolean[i][j]，则将该位置修改为0。

  3. 用第一行/列记录整行/列是否出现0，但是需要注意记录第0行第0列是否出现0。后续遍历从第1行/列开始即刻。

LC 字母异位词分组

给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

字母异位词 是由重新排列源单词的字母得到的一个新单词，所有源单词中的字母通常恰好只用一次。

示例 1:

输入: strs = [“eat”, “tea”, “tan”, “ate”, “nat”, “bat”]
输出: [[“bat”],[“nat”,”tan”],[“ate”,”eat”,”tea”]]

直接hashmap ，key为字符串的字典序，value为异位set

LC 无重复字符的最长子串

给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

数组长度容易出错，使用linkedList作为滑动窗口

  1. 之前未现过该字符时，加入窗口
  2. 当之前出现过该字符时，从窗口中移除首字符。保证滑动窗口内字符的唯一性

LC 最长回文子串

给你一个字符串 s，找到 s 中最长的回文子串。

如果字符串的反序与原始字符串相同，则该字符串称为回文字符串。

暴力就算了，马拉车也算了

常见的有中心扩散，但是涉及到单双问题，麻烦。不如dp

dp递推条件

boolean[i][j] 储存从index i 到 index j 是否为回文串 如果 chars[i]==chars[j] and boolean[i+1[j-1]，证明找到一条更长的子回文串

代码逻辑需要注意的是，长的子串依赖短子串，比如 dp[0][3] 依赖dp[1][2] ，所以需要倒序，也就是

  • 1-0
  • 2-0，2-1，
  • 3-0，3-1，3-2

然后在chars[i]==chars[j]情况下，如果 (j - i +1)<=3 则说明是aba或者 aa情况，直接为true

LC 递增的三元子序列

给你一个整数数组 nums ，判断这个数组中是否存在长度为 3 的递增子序列。

如果存在这样的三元组下标 (i, j, k) 且满足 i < j < k ，使得 nums[i] < nums[j] < nums[k] ，返回 true ；否则，返回 false 。

这道题没思路，看完解答才发现如此简单:

  • 如果当前数字比最小值还小，我们就更新最小值
  • 如果当前数字大于最小值并且小于第二小的值，我们就更新第二小的值
  • 如果当前数字大于第二小的值，说明找到了递增的三元子序列，直接返回true。
